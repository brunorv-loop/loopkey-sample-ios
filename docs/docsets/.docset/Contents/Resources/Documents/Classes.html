  • LKDeviceCommunicator

    This class is responsible for managing a LKDevice communication. It is used for sending some type of message to any LKDevice. An instance of this class should be generated for the device that is going to be communicated with once a new communication is required.

    The steps to use this instace are:

    1. Initialize the instance with the device that should communicated, a user responsible for the communication and an delegate who implements the LKDeviceCommunicatorProtocol.
    2. Call communicateWithType everytime a new communication is going to be made with the device.

  • LKCommunicationManager

    This class is the main communication manager from the device with a LoopKey. It can be used for discovering a device or to send a messsage. This is an abstract class and a concrete implementation should be provided according to the communication method used to send messages to the LK Device (e.g. BLE).

    Warning

    This class should never be instantiated. It must be subclassed.

    Using the service for discovering a device:

    1. The class has to implement the LKDiscoveryProtocol for listening to response events.
    2. The class can subscribeForNotifications to discover when has changed its state from reachable to unreachable and vice-versa. In that case that class has to unsubscribeForNotifications when it wants to stop listenning to these events.
    3. The class can get immediately the current list of visible devices by calling getVisibleDevices.

    Using the service for connecting with a device:

    1. If a a rollCount is need the user can cal readRollCount first.
    2. To send a message the user can call send.
