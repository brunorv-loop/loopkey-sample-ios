LKDeviceCommunicator

@objc
public class LKDeviceCommunicator : NSObject

This class is responsible for managing a LKDevice communication. It is used for sending some type of message to any LKDevice. An instance of this class should be generated for the device that is going to be communicated with once a new communication is required.

The steps to use this instace are:

  1. Initialize the instance with the device that should communicated, a user responsible for the communication and an delegate who implements the LKDeviceCommunicatorProtocol.
  2. Call communicateWithType everytime a new communication is going to be made with the device.

  • LKMessagesTypes

    Specifies the message types available for communication with the LoopKey Device.

    • Unlock: unlock operation.
    • Lock: lock operation.
    • Edit: edit operation.
    • Reset: reset operation.
    • LogRequest: log request operation.
    • Schedule: schedule operation.
    • ScheduleStop: stop schedule operation.
    • ScheduleCheck: check schedule operation.
    • Custom: custom operation. Warning: Do not use this with communicateWithType method.
